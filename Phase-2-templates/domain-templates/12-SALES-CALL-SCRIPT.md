## 🎯 TEMPLATE

```
ROLE: You are an elite sales conversation architect with 15+ years building scripts for SaaS, enterprise, and high-ticket sales.
Your specialty: Creating natural, non-scripted frameworks that guide reps to discovery and close.

CONTEXT: I'm selling [PRODUCT/SERVICE] to [TARGET_BUYER] at [COMPANY_TYPE] companies.
The typical sales cycle is [CYCLE_LENGTH] and average deal size is [DEAL_SIZE].
Common objections include: [OBJECTION_1], [OBJECTION_2], [OBJECTION_3].

TASK: Create a structured sales call script/framework for [CALL_TYPE: Discovery/Demo/Closing] that guides the conversation while feeling natural.

REQUIREMENTS:
- Opening (30 seconds): Pattern interrupt + credibility establish + agenda set
- Discovery (10-15 min): 5-7 strategic questions using SPIN methodology
- Value presentation (5-10 min): Connect features to SPECIFIC prospect pain points
- Objection handling: Pre-loaded responses for top 3 common objections
- Close: 3 tiered asks (soft → medium → hard commitment)
- Each section must have "IF prospect says X, THEN you say Y" branches
- Include pause points (where rep should STOP talking)
- Conversation checkpoints: How to know when to move to next phase

CONSTRAINTS:
- NO robotic "How are you today?" openings
- NO feature dumping before pain discovery
- NEVER ask "What keeps you up at night?" or similar clichés
- NO more than 3 minutes of monologue at any point
- Never make the prospect feel interrogated (balance questions with insights)

AUDIENCE: [BUYER_PERSONA_TITLE] with [BUYER_MOTIVATION: cost savings/growth/risk reduction]

OUTPUT FORMAT:
Section 1: PRE-CALL PREP (3 things to research)
Section 2: OPENING SCRIPT (with variations)
Section 3: DISCOVERY QUESTION MAP (Primary + follow-ups)
Section 4: VALUE BRIDGE (feature → pain → outcome)
Section 5: OBJECTION RESPONSES (acknowledge → reframe → resolve)
Section 6: CLOSING PROGRESSION (soft ask → medium ask → hard ask)
Section 7: POST-CALL ACTION ITEMS

SELF-CHECK BEFORE RESPONDING:
Rate your script:
- Natural conversation flow (1-3):
- Specificity to buyer persona (1-3):
- Clear progression to close (1-3):
If total < 7 — add more buyer-specific branches.
```

## 🎨 REAL EXAMPLES

**Example 1: SaaS Discovery Call**
```
OPENING:
"Hi [Name], before we dive in — quick context: I saw [Company] just expanded into [Market]. 
That's exactly the trigger point where [Problem] usually becomes critical. 
Sound familiar, or are you in a different situation?"

DISCOVERY QUESTION 1:
"When you're onboarding [Number] new [Role]s monthly, how are you currently handling [Process]?"
[If vague] → "Walk me through the last time someone had to [Task]. What does that look like?"
```

**Example 2: Closing Call**
```
SOFT ASK:
"Based on what we've covered — does this feel like it would solve the [Pain] we discussed?"

MEDIUM ASK:
"If we could get you started with a pilot in [Timeframe], would that work for your team's schedule?"

HARD ASK:
"Should we get the paperwork started for a [Date] kickoff?"
```

## 🔧 VARIATIONS

```
Approach: Honest, direct, non-desperate
```

## ✅ QUALITY CHECKLIST

- [ ] Opening references something SPECIFIC to prospect (not generic)
- [ ] Discovery questions follow logical pain → implication → need flow
- [ ] Each question has 2-3 natural follow-up branches
- [ ] Value presentation connects to pains discovered (not generic)
- [ ] Objection responses acknowledge before pivoting
- [ ] Close has 3 distinct commitment levels
- [ ] Script includes explicit "pause and listen" markers

---

## 💡 PRO TIPS

- **Use the "Teach, Don't Tell" technique**: Share one insight about their industry before asking anything
- **Apply the "Labeling" technique**: "It sounds like [X] is a bigger priority than [Y] right now" → gets them to confirm or correct
- **Silence is a tool**: Mark 3-second pause points after key questions
- **Use "For example" liberally**: Makes abstract benefits concrete
- **The best objection handling is prevention**: Address concerns in value section before they raise them
- **Close on a spectrum**: Give prospects choice between small/medium/big commitments

2. **Call 6-10**: A/B test opening variations
3. **Call 11-20**: Refine objection responses based on real pushback

5. **Quarterly**: Full script refresh based on win/loss analysis
